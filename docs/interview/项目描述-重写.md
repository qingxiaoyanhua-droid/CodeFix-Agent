智能代码修复 Multi-Model Agent 系统 / 项目技术负责人

■ 项目概述
通用大模型在代码修复场景面临上下文短、推理慢、成本高的问题。本项目设计 Compile-First
Multi-Model Agent 系统：主模型通过 AST 解析代码结构，动态路由到本地小模型（1.5B）生成
修复或云端大模型（32B）提供裁判反馈，配合 RAG + 双层记忆 + GRPO 训练，pass@1 54%→87%，
延迟降低 44%，大模型调用减少 60%。

■ 核心工作

（1）Multi-Model Agent 架构
    主模型（ReAct 调度中枢）+ 小模型（本地代码生成）+ 大模型（云端裁判反馈）三层协作。
    Compile-First 验证：编译器+测试用例作 ground truth，零幻觉；未通过则大模型介入，最多
    3 轮。60% case 首轮直接通过。Ruff 静态分析兜底：编译通过后自动检查代码风格与可读性，
    有问题则触发轻量优化 pass，无需大模型参与。参考 DECENTMEM 双池设计（E-pool 成功轨迹 / X-pool 候选
    策略），路由器按相似度动态选择记忆池。

（2）三层记忆系统
    L1 上下文压缩 + L2 反思记忆（失败经验，时间衰减权重）+ L3 技能记忆（成功 SOP）。
    质量门槛：仅编译+测试双通过的修复写入 E-pool，防止脏数据；每周清理低权重碎片。

（3）RAG + GRPO
    BM25+FAISS 混合检索（596 条），RRF 融合 + Cross Encoder 重排，延迟 <200ms，准确率 85%+，
    pass@1 额外提升 5%。GRPOTrainer 多粒度奖励（格式+语法+AST+测试执行），防止 Reward hacking。

（4）代码修复引擎 + 安全沙箱
    AST 函数级定位（字节偏移，无行号问题）+ 7 维度 bug 检测 + diff 验证。
    6 层沙箱（语言白名单+危险模式+subprocess 隔离+临时目录+资源限制+退出码验证）。
    FastAPI + API Key 鉴权 + 滑动窗口限流（30 req/min/key）+ JSONL 审计日志。

■ 项目成果
pass@1 54% → 87%（+33pp），验证 RAG+RLHF 在代码修复场景的有效性，L2 命中重试成功率+18%。
