## 🚀 部署步骤

### Step 1: 环境准备

```bash
# 1. 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 2. 升级 pip
pip install --upgrade pip

# 3. 安装 PyTorch (根据 CUDA 版本)
# CUDA 11.8
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118

# 或 CUDA 12.1
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
```

### Step 2: 解压项目

```bash
# 解压项目包
tar -xzf repobugfix_complete_project.tar.gz
cd repobugfix-agent

# 检查文件
ls -lh
```

### Step 3: 安装依赖

```bash
# 安装项目依赖
pip install -r requirements.txt

# requirements.txt 包含：
# transformers>=4.38.0
# trl>=0.8.0
# datasets>=2.14.0
# peft>=0.8.0
# accelerate>=0.25.0
# faiss-cpu>=1.7.4
# openai>=1.0.0
```

### Step 4: 配置多 GPU

```bash
# 检查 GPU
nvidia-smi

# 应该看到 2 张 A100

# 设置可见 GPU
export CUDA_VISIBLE_DEVICES=0,1
```

### Step 5: 配置 API Key

```bash
# 设置 DashScope API Key（用于 LLM Critic）
export DASHSCOPE_API_KEY="sk-your-api-key"

# 如果没有 API Key，可以禁用 LLM Critic
# 修改 grpo_codefix_train_with_critic.py:
# USE_LLM_CRITIC = False
```

### Step 6: 运行 GRPO 训练（单卡）

```bash
# 使用单张 A100 训练
CUDA_VISIBLE_DEVICES=0 \
python grpo_codefix_train_with_critic.py > grpo_training.log 2>&1

# 监控训练
tail -f grpo_training.log
```

**预期时长**: 20-30 分钟（187 条数据）

### Step 7: 运行 GRPO 训练（双卡并行）

```bash
# 使用 accelerate 进行分布式训练
accelerate config

# 配置：
# - This machine: 2 GPUs
# - GPU _ids: 0,1
# - Mixed precision: fp16

# 启动训练
accelerate launch grpo_codefix_train_with_critic.py > grpo_training.log 2>&1

# 监控
tail -f grpo_training.log
```

**预期加速**: 1.5-1.8x（相比单卡）

## 🔧 常见问题

### Q1: 显存不足怎么办？

**A**: 减小 batch size 或 num_generations

```python
# 修改 grpo_codefix_train_with_critic.py
BATCH_SIZE = 1  # 从 2 改为 1
NUM_GENERATIONS = 2  # 从 4 改为 2
```

### Q2: 没有 API Key 怎么运行？

**A**: 禁用 LLM Critic

```python
# 修改 grpo_codefix_train_with_critic.py
USE_LLM_CRITIC = False
```

### Q3: 如何使用本地模型？

**A**: 修改模型路径

```python
# 修改 MODEL_NAME
MODEL_NAME = "/path/to/local/Qwen2.5-Coder-7B"
```

### Q4: 如何保存检查点？

**A**: 训练会自动保存

```
outputs/codefix-grpo-llm-critic/
├ checkpoint-50/
├ checkpoint-100/
└ final/
```

## 🎯 推理测试

### 加载训练好的模型

```python
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import PeftModel

# 加载基座模型
model = AutoModelForCausalLM.from_pretrained(
    "/data/models/model_cache/Qwen/Qwen2.5-Coder-7B-Instruct",
    torch_dtype=torch.float16,
    device_map="auto"
)

# 加载 LoRA adapter
model = PeftModel.from_pretrained(
    model,
    "outputs/codefix-grpo-llm-critic/final"
)

tokenizer = AutoTokenizer.from_pretrained(
    "/data/models/model_cache/Qwen/Qwen2.5-Coder-7B-Instruct"
)

# 推理
prompt = """Bug: Fix the addition function

Buggy Code:
```python
def add(a, b):
    return a - b
```

Fix the bug."""

inputs = tokenizer(prompt, return_tensors="pt").to(model.device)
outputs = model.generate(**inputs, max_new_tokens=256)
print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```
