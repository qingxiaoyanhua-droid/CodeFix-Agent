```python
class CodeFixModel:
    def __init__(self, base_path, lora_path):
        self.device = torch.device("cuda")
        self.tokenizer = AutoTokenizer.from_pretrained(base_path, trust_remote_code=True)
        self.tokenizer.pad_token = self.tokenizer.eos_token
        
        base_model = AutoModelForCausalLM.from_pretrained(
            base_path, torch_dtype=torch.float16, trust_remote_code=True
        ).to(self.device)
        
        self.model = PeftModel.from_pretrained(base_model, lora_path)
        self.model.eval()

    def generate(self, prompt, num_samples=5):
        inputs = self.tokenizer(prompt, return_tensors="pt").to(self.device)
        outputs = []
        for _ in range(num_samples):
            with torch.no_grad():
                output = self.model.generate(
                    **inputs, max_new_tokens=256, do_sample=True,
                    temperature=0.7, top_p=0.9, pad_token_id=self.tokenizer.eos_token_id,
                )
            result = self.tokenizer.decode(output[0], skip_special_tokens=True)
            outputs.append(result[len(prompt):].strip())
        return outputs

class RAGSearcher:
    def __init__(self, index_dir):
        with open(f"{index_dir}/bm25.pkl", 'rb') as f:
            self.bm25 = pickle.load(f)
        with open(f"{index_dir}/knowledge_base.json", 'r', encoding='utf-8') as f:
            self.knowledge_base = json.load(f)

    def search(self, query, top_k=2):
        query_tokens = query.lower().split()
        scores = self.bm25.get_scores(query_tokens)
        top_indices = np.argsort(scores)[::-1][:top_k]
        return [self.knowledge_base[idx] for idx in top_indices]
```

### 5. GRPO 训练尝试 (多次失败)

#### 5.1 训练方案设计

**架构**:
```
Policy Model (1.5B) → GPU 0 (生成代码)
                      ↓
Reward Model (7B)   → GPU 1 (计算奖励，使用 perplexity)
```

**训练配置**:
| 参数 | 值 |
|------|-----|
| BATCH_SIZE | 1 |
| NUM_GENERATIONS | 2-4 (每 prompt 生成候选数) |
| LEARNING_RATE | 5e-6 |
| NUM_EPOCHS | 1 |
| MAX_LENGTH | 512 |
| GRADIENT_ACCUMULATION_STEPS | 4 |
| LoRA r | 8 |
| LoRA alpha | 16 |
| LoRA target_modules | ["q_proj", "k_proj", "v_proj", "o_proj"] |
| LoRA dropout | 0.05 |

**量化方案**:
```python
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_use_double_quant=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.float16
```

#### 5.2 GRPO 核心逻辑

```python
class GRPOTrainer:
    def grpo_loss(self, log_probs, rewards):
        if len(rewards) < 2:
            return torch.tensor(0.0, device=self.policy_device, requires_grad=True)
        
        # 转换为 tensor
        rewards = torch.tensor(rewards, dtype=torch.float32, device=self.policy_device)
        log_probs = torch.tensor(log_probs, dtype=torch.float32, device=self.policy_device)
        
        # 组内标准化奖励
        mean_reward = rewards.mean()
        std_reward = rewards.std() + 1e-8
        normalized_rewards = (rewards - mean_reward) / std_reward
        
        # GRPO 核心损失: policy gradient with normalized rewards
        loss = -torch.mean(log_probs * normalized_rewards)
        return loss
    
    def train(self):
        for batch in self.dataloader:
            for prompt in batch['prompt']:
                # 生成多个候选
                completions, log_probs = self.generate_completions(prompt)
                
                # 计算每个候选的奖励 (用 7B 模型的 perplexity)
                rewards = []
                for completion in completions:
                    reward = self.reward_model.compute_reward(prompt, completion)
                    rewards.append(reward)
                
                # GRPO 损失 + 梯度更新
                loss = self.grpo_loss(log_probs, rewards)
                loss = loss / GRADIENT_ACCUMULATION_STEPS
                loss.backward()
                
                if (batch_idx + 1) % GRADIENT_ACCUMULATION_STEPS == 0:
                    self.optimizer.step()
                    self.optimizer.zero_grad()
```

**奖励计算**:
```python
class RewardModel:
    def compute_reward(self, prompt, completion):
        full_text = f"{prompt}\n{completion}"
        inputs = self.tokenizer(full_text, return_tensors="pt").to(self.device)
        
        with torch.no_grad():
            outputs = self.model(**inputs, labels=inputs["input_ids"])
            loss = outputs.loss
        
        # perplexity 越低，奖励越高
        reward = -loss.item() + 1e-6
        return reward
```

## ⚠️ 关键问题总结

### 问题 1: HuggingFace 依赖版本冲突

**冲突链**:
```
huggingface_hub 1.5.0
  └─ datasets 4.6.1 要求 huggingface-hub>=0.25.0 ✓
  
transformers 4.35.2
  └─ 要求 huggingface-hub>=0.16.4
  └─ 实际安装 0.17.3 (降级后)
  
sentence-transformers 5.2.3
  └─ 要求 huggingface-hub>=0.20.0 ✗ (实际 0.17.3)
  └─ 要求 transformers>=4.41.0 ✗ (实际 4.35.2)
```

**建议解决方案**:
```bash
# 使用兼容版本组合
pip install transformers==4.41.0 tokenizers==0.19.0 huggingface-hub==0.24.0 datasets==2.20.0
```

### 问题 2: 显存管理

**现状**:
- GPU 0: 被多个进程占用 ~18GB (用户自己的进程)
- GPU 1: 被其他用户进程占用 ~27GB (无法 kill)

**可用方案**:
1. **单 GPU 训练**: 两个模型都放 GPU 0，但需要极小的 batch size
2. **CPU offload**: 用 `accelerate` 库将部分计算 offload 到 CPU
3. **等空闲时段**: 其他用户释放 GPU 1 后再训练
4. **只加载一个模型**: 先用 1.5B 模型做 SFT，不训练 GRPO

### 问题 3: Qwen2 Tokenizer 兼容性

**根本原因**: transformers 4.35.2 不支持 `Qwen2Tokenizer` 类

**解决方案**:
1. 升级 transformers >= 4.37.0 (Qwen2 支持版本)
2. 手动复制 `tokenization_qwen.py` 到模型目录
3. 使用 Qwen1.5 系列 (而非 Qwen2.5)

### 问题 4: 模型路径特殊字符

**问题**: ModelScope 下载时 `.` 被替换为 `___`

**修复**:
```bash
# 创建软链接
```

## 💡 经验总结与建议

### 1. 依赖管理

**教训**: HuggingFace 生态版本兼容性复杂，随意升级/降级会导致连锁冲突

**建议**:
```bash
# 1. 事前锁定版本
pip freeze > requirements.txt

# 2. 使用虚拟环境
conda create -n grpo_env python=3.10
conda activate grpo_env

# 3. 安装顺序很重要
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
pip install transformers==4.41.0 datasets==2.20.0 peft==0.11.1 trl==0.9.4
pip install accelerate bitsandbytes
```

### 2. 显存管理

**教训**: 多用户共享 GPU 时，必须提前确认可用显存

**建议**:
```bash
# 1. 查看显存占用
nvidia-smi
nvidia-smi --query-compute-apps=pid,process_name,used_memory --format=csv

# 2. 杀掉自己的无用进程
pkill -f python  # (仅限自己的进程)

# 3. 设置显存碎片整理
export PYTORCH_ALLOC_CONF=expandable_segments:True
```

### 3. 模型加载

**教训**: 离线环境需要 `local_files_only=True`，且模型目录必须完整

**建议**:
```python
# 正确加载方式
tokenizer = AutoTokenizer.from_pretrained(
    "/path/to/model",
    trust_remote_code=True,
    local_files_only=True,   # 强制离线
    padding_side="right"
)

model = AutoModelForCausalLM.from_pretrained(
    "/path/to/model",
    torch_dtype=torch.float16,
    local_files_only=True,
    device_map="auto"
)
```

### 4. 数据集处理

**教训**: 在线加载数据集可能因网络问题失败，优先下载本地副本

**建议**:
```bash
# 1. 提前下载数据集
huggingface-cli download livecodebench/code_generation_lite --repo-type dataset

# 2. 本地加载
from datasets import load_from_disk
dataset = load_from_disk("/path/to/local/dataset")
```

### 5. 调试策略

**教训**: 复杂训练流程应逐步验证每个组件

**建议流程**:
```
1. 测试模型能否加载         ✅ (test_model.py)
2. 测试单条生成             ✅ (test_model.py 成功)
3. 测试数据集加载           ✅ (转换脚本成功)
4. 测试奖励计算             ❌ (OOM)
5. 测试完整训练流程         ❌ (未成功)
```
