#### 关键代码片段
```python
class CodeFixModel:
    def __init__(self, base_path, lora_path):
        self.device = torch.device("cuda")
        self.tokenizer = AutoTokenizer.from_pretrained(base_path, trust_remote_code=True)
        self.tokenizer.pad_token = self.tokenizer.eos_token
        
        base_model = AutoModelForCausalLM.from_pretrained(
            base_path, torch_dtype=torch.float16, trust_remote_code=True
        ).to(self.device)
        
        self.model = PeftModel.from_pretrained(base_model, lora_path)
        self.model.eval()

    def generate(self, prompt, num_samples=5):
        inputs = self.tokenizer(prompt, return_tensors="pt").to(self.device)
        outputs = []
        for _ in range(num_samples):
            with torch.no_grad():
                output = self.model.generate(
                    **inputs, max_new_tokens=256, do_sample=True,
                    temperature=0.7, top_p=0.9, 
                    pad_token_id=self.tokenizer.eos_token_id,
                )
            result = self.tokenizer.decode(output[0], skip_special_tokens=True)
            outputs.append(result[len(prompt):].strip())
        return outputs
```

## 💡 经验总结与建议

### 1. 依赖管理
- **优先使用 `requirements.txt` 锁定版本**，避免运行时冲突
- HuggingFace 生态更新频繁，建议定期检查兼容性
- 推荐稳定版本组合：
  ```
  transformers>=4.36.0
  tokenizers>=0.15.0
  huggingface_hub>=0.25.0
  datasets>=2.13.0
  ```

### 2. 显存管理
- **训练前检查 GPU 占用情况**：`nvidia-smi`
- 多用户环境下无法强制 kill 他人进程
- 建议：
  - 优先使用单 GPU 方案
  - 采用 4bit/8bit 量化降低显存需求
  - 设置 `PYTORCH_ALLOC_CONF=expandable_segments:True` 避免内存碎片

### 3. 离线部署
- 服务器可能无法访问外网，需提前准备：
  - 模型权重本地下载
  - 依赖包离线安装
  - 使用 `local_files_only=True` 参数
  - 准备纯离线脚本（不依赖在线 API）

### 4. 模型选择
- Qwen2.5-Coder 系列需要较新的 transformers 版本
- 1.5B 模型显存占用低（~3GB），适合快速验证
- 7B 模型性能更好但显存占用高（~14GB FP16）

### 5. 数据安全
- 重要数据集多重备份
- 避免依赖在线加载，优先本地缓存
- 转换脚本应有容错机制
